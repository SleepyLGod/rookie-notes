# Distributed Systems

This section contains notes on distributed systems and classic Google system papers.

Suggested reading order:

1. [CAP theorem](cap-theorem.md): first clarify the boundaries between consistency, availability, and partition tolerance.
2. [Google File System](gfs.md) and [Google MapReduce](mapreduce.md): understand the storage and computation foundations of large-scale data processing.
3. [Google Bigtable](bigtable.md): finally read the distributed storage model for structured data.

Content status: these notes are mainly paper-reading notes and concept summaries. Some real-world systems have evolved for many years since the papers were written. For current product behavior, check the official documentation again.
